# models/arch/

FPGA AI Suite 2026.1.1 architecture files (`.arch`) for the Agilex 3 (`AGX3`) family, copied
**unmodified** from the installed `alterafpga/fpgaaisuite:2026.1.1-quartus` image at
`/opt/altera/fpga_ai_suite/ubuntu/dla/example_architectures/` (confirmed present there — see
`docs/toolchain.md`). Per issue #6: "v1 arch files should be the vendor examples, unmodified" — do
not hand-edit these to chase a prettier estimator number; a follow-on issue owns generating a
tuned/custom AGX3 arch.

This AI Suite release ships exactly three `AGX3_*.arch` files (versus, e.g., a dozen-plus for the
bigger AGX7 family) — there is no `*_LayoutTransform*` or `*_Ddrfree*` AGX3 variant at all. All
three are `16x16, num_interleaved_features=12` (the same PE-array size); they differ only in which
activation/pooling/softmax modules are enabled:

| File | Long-form name (vendor `README.txt`) | Role used in this project |
|---|---|---|
| `AGX3_Performance.arch` | `AGX3/16x16_i12x1_fp12agx_sb8192_poolk4_actk16_prelu_rclamp_sig_softmaxk1` | **Only usable arch for INT8**, and PH0's stand-in for PLAN §9 PH3 config (b) "external-memory" |
| `AGX3_Small_NoSoftmax.arch` | `AGX3/16x16_i12x1_fp12agx_sb8192_poolk1_actk16_clamp` | Smallest-footprint file; PH0's stand-in for PLAN §9 PH3 config (a) "DDR-free-style small footprint" — see caveat below |
| `AGX3_Small_Softmax.arch` | `AGX3/16x16_i12x1_fp12agx_sb8192_poolk4_actk16_clamp_softmaxk1` | Small-footprint + softmax; kept for completeness, same INT8 caveat as `Small_NoSoftmax` |

## Two findings from actually running the estimator against these files (issue #6)

**1. Only `AGX3_Performance.arch` can compile an INT8-quantized (NNCF FakeQuantize) graph at all.**
`dla_compiler --fanalyze-performance` refuses both `Small` files with: *"Quantized graphs with
FakeQuantize require Scale enabled and Round Clamp activation in the architecture"* — and indeed
only `AGX3_Performance.arch` sets `pe_array.enable_scale: true` and
`activation.enable_round_clamp: true`; neither `Small` file sets either. So there is no vendor AGX3
arch that is simultaneously small-footprint *and* INT8-capable — every INT8 estimate in this
project's `results/*.json` uses `AGX3_Performance.arch`. This is a real, verified constraint, not a
guess: reproduce with
`dla_compiler --fanalyze-performance --march models/arch/AGX3_Small_NoSoftmax.arch --network-file models/ir/ad-toycar/int8/ad-toycar.xml --fplugin HETERO:FPGA ...`
(see `docs/toolchain.md` for the full invocation).

**2. None of the three files is actually "DDR-free" (MIF-resident weights).** All three declare a
`dma { ddr_addr_width: 32; ... }` block and a `stream_buffer_depth`, and none has the
`config_network` block the AI Suite's own `--gen-arch` help text associates with genuine DDR-free
architectures (contrast the AGX7 family's dedicated `*_Ddrfree*` files, which AGX3 has no
equivalent of in this release). Confirmed empirically too: even the smallest model in this
project's corpus (`ad-toycar`, 267 KB of INT8 weights — PLAN §5 calls this **M20K-resident,
DDR-free**) shows `DDR FILTER READS REQUIRED` on *every single inference* in the estimator output
against `AGX3_Performance.arch` (`results/ph0_ad-toycar-*.json`), i.e. weights are being re-streamed
from external memory every inference, not held resident on-chip. So "config (a) DDR-free-style
small footprint" in this issue's deliverable list is approximated by the smaller of the two
`Small_*` files (by module count) **in name only** — getting a real DDR-free AGX3 configuration
will need either a custom arch (out of PH0's "vendor examples, unmodified" scope) or the
`--skip-mif-gen`/system-integration path in PLAN §9 PH3, not just picking a different example file.
See `results/reports/ph0_estimator.md` for what this does to the §5 planning numbers.
